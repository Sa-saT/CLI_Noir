## 2. レベル別コマンド拡張表

### Level 5: 情報屋 — パイプ・テキスト処理（LPIC 103.2 / 103.4）

「1つのコマンドで終わらせず、流れ作業で情報を捌く」段階。

| コマンド | 実PCでの意味 | ゲーム内での対応（提案） |
|---|---|---|
| `\|`（パイプ） | 前のコマンドの出力を次のコマンドの入力へ渡す | 捜査を流れ作業にする。「読む→絞る→数える」を1行で行う情報屋の技 |
| `head` | ファイルの先頭N行を表示 | 分厚い調書の冒頭だけ確認する（全部読む必要はない、と学ぶ） |
| `tail` | ファイルの末尾N行を表示 | 日誌・ログの「最新の記録」を確認する |
| `tail -f` | 追記をリアルタイム監視 | 盗聴ログを張り込み監視する（ライブ表示演出） |
| `wc` | 行数・単語数・バイト数を数える | 証拠の分量を数える。「容疑者リストは全部で何人？」 |
| `cut` | 行から特定の列（フィールド）を切り出す | 名簿から「電話番号の列だけ」抜き出す |
| `paste` | 複数ファイルの行を横に結合 | 破られたメモを貼り合わせて復元する |
| `tr` | 文字の置換・削除 | 換字式暗号（シーザー暗号など）を解読する |
| `sed` | ストリーム編集（置換・削除・抽出） | 改ざんされた文書を元の記述に修正する |
| `diff` | 2ファイルの差分を表示 | 原本と偽造文書を見比べ、改ざん箇所を特定する |
| `nl` | 行番号を付けて表示 | 証拠文書に整理番号を振る |
| `tee` | 出力を画面とファイルの両方へ書く | 確認しながら同時に記録も残す（捜査メモの複写） |
| `xargs` | 標準入力を次のコマンドの引数に変換 | 見つけた証拠リストをまとめて次の処理にかける |

**ゲーム設計のポイント:**
「数万行のログファイル」を配置し、`cat` で全部読むのは事実上不可能な状況を作る。
`grep \| sort \| uniq -c` でしか現実的に解けない Mission により、パイプの**必要性**を体感させる。

### Level 6: 監視者 — プロセス（LPIC 103.5）

「PC の中では常に何かが動いている」ことを体験する段階。仮想プロセステーブルを evaluator に持たせる。

| コマンド | 実PCでの意味 | ゲーム内での対応（提案） |
|---|---|---|
| `ps` | 実行中プロセスの一覧表示 | 「今この建物の中で動いている者」を一覧する。PID＝容疑者番号 |
| `ps aux` | 全ユーザーの全プロセスを詳細表示 | 建物全体の人物・挙動の詳細リスト |
| `top` | プロセスをリアルタイム監視 | 監視カメラモニター（CPU 使用率＝怪しい動きの激しさ） |
| `kill <PID>` | プロセスに終了シグナルを送る | 盗聴プログラムを停止させる（＝確保） |
| `pgrep` / `pkill` | 名前でプロセスを検索/終了 | 名前から容疑プロセスを割り出して止める |
| `jobs` / `fg` / `bg` | ジョブの一覧/前面/背面切替 | 並行して進めている捜査タスクの切り替え |
| `free` / `uptime` | メモリ使用量 / 稼働時間の確認 | 建物の稼働状況・「いつから動いているか」を調べる |

**ゲーム設計のポイント:**
`kill` は実PCでも「プロセスを止める」以上のことはしない、と正しく学ばせる。
正規プロセス（`clock`, `mailbox` など）を kill すると警告＋巻き戻し（即失敗にしない方針を維持）。

### Level 7: 潜入捜査官 — 権限・ユーザー（LPIC 104.5 / 107.1 / 110.2）

Level 分けで既出の `chmod` / `chown` をここで本格導入し、ユーザー概念まで広げる。

| コマンド / 概念 | 実PCでの意味 | ゲーム内での対応（提案) |
|---|---|---|
| `chmod` | ファイル権限の変更 | 鍵のかかった証拠ファイルを解錠する（既存案の具体化） |
| `chmod +x` | 実行権限の付与 | `case_file.sh` が「実行できない」状態から自力で解除する |
| `chown` | 所有者の変更 | 証拠品の管理責任者を移す |
| `umask` | 新規ファイルのデフォルト権限を決める | 「作るメモに最初からかかる鍵」の設定 |
| パーミッション `rwx` | 読み/書き/実行の3権限 × 所有者/グループ/他人 | 「読める鍵・書ける鍵・開ける鍵」×「自分・仲間・部外者」 |
| `whoami` / `id` | 現在のユーザー名 / UID・所属グループ確認 | 変装中の「今の自分は誰か」を確認する |
| `su <user>` | 別ユーザーに切り替え | 潜入捜査のための変装。パスワード＝合言葉 |
| `who` / `w` | ログイン中ユーザーの確認 | 「今この建物に誰が入っているか」を調べる |
| `useradd` / `passwd` | ユーザー作成 / パスワード設定 | 協力者の身分証を発行する（Phase3 候補） |
| `sudo` | 管理者権限で実行 | **原則封印**（denylist 維持）。最終 Mission のイベント演出限定で検討 |

**ゲーム設計のポイント:**
`ls -l` の `rwxr-xr--` 表示を「鍵の刻印」として読む訓練を Mission 内に組み込む。
LPIC で頻出の数値モード（`chmod 755` 等）は、記号モード習得後の加点要素にする。

### Level 8: 証拠管理官 — アーカイブ・リンク・鑑識（LPIC 103.3 / 104.6）

| コマンド | 実PCでの意味 | ゲーム内での対応（提案） |
|---|---|---|
| `tar -xf` / `tar -cf` | アーカイブの展開 / 作成 | 封印された証拠箱を開ける / 証拠を箱詰めして保管する |
| `gzip` / `gunzip` | 圧縮 / 展開 | 圧縮された押収データを復元する |
| `zip` / `unzip` | zip 形式の圧縮/展開 | 海外から届いた証拠品（形式が違う箱）を開ける |
| `ln -s` | シンボリックリンク作成 | 「本物の場所を指す案内板」を立てる |
| `ln` | ハードリンク作成 | 同一の証拠品に別名を付ける（案内板との違いを学ぶ） |
| `file` | ファイル種別の判定 | 鑑識にかける。「拡張子は .txt だが中身は画像」を見抜く |
| `which` / `whereis` | コマンドの実体パスを調べる | 道具（コマンド）の保管場所を調べる |
| `locate` | データベースから高速ファイル検索 | 事件記録データベースへの照会（`find` との使い分けを学ぶ） |
| `du` / `df` | ディレクトリ使用量 / ディスク空き容量 | 倉庫の使用状況・空き確認 |

### Level 9: 追跡者 — ネットワーク（LPIC 109.x / 110.1）

Mission3 の `ssh` 体験を発展させ、「接続する前に調べる」を学ぶ。

| コマンド | 実PCでの意味 | ゲーム内での対応（提案） |
|---|---|---|
| `ping` | 相手ホストへの疎通確認 | 「その場所は今も存在するか」生存確認の電話を鳴らす |
| `ip addr` | 自分の IP アドレス確認 | 自分の現在地（住所）を確認する |
| `ss` | 開いているポート・接続の確認 | 「建物のどの扉が開いているか」を調べる |
| `dig` / `host` | ドメイン名から IP を引く（DNS） | 通称（ホスト名）から実際の住所（IP）を割り出す |
| `hostname` | 自ホスト名の確認 | 今いる拠点の名前を確認する |
| `traceroute` | 相手までの経路を表示 | 犯人サーバーまでの追跡ルートを地図に描く |
| `curl` | HTTP リクエスト送信（既存・mock 限定） | 本部データベース API への照会（既存仕様を拡張） |

**ゲーム設計のポイント:**
「`dig` で IP を突き止める → `ping` で生存確認 → `ssh` で突入」という実務と同じ順序を
Mission の必須フローにする。DNS の概念（名前→住所の解決）を追跡劇として体験させる。

### Level 11: 参謀 — シェルスクリプト基礎（LPIC 105.1 / 105.2）

「手作業の捜査を手順書（スクリプト）に落とす」最終段階。これまで実行するだけだった
`case_file.sh` の中身を、自分で書く側に回る。

| コマンド / 構文 | 実PCでの意味 | ゲーム内での対応（提案） |
|---|---|---|
| `sh <file>` | シェルスクリプトの実行 | 手順書どおりに捜査を自動実行する（`case_file.sh` の一般化） |
| 変数 `NAME=値` / `$NAME` | 値に名前を付けて保存・参照する | 捜査メモに名前を付けて持ち歩く（`SUSPECT="..."`） |
| `$?` | 直前コマンドの終了コード（0=成功） | 「さっきの捜査は成功したか？」への部下の返事 |
| `if` / `then` / `fi` | 条件分岐 | 「証拠が見つかったら報告する」を手順書に書く |
| `for` / `do` / `done` | 繰り返し処理 | 全ファイルへの聞き込みを自動化する |

| `read` | 標準入力から値を受け取る | 手順書の途中で相棒に質問する |

### コマンドではない学習要素（概念・記号・キー操作）

LPIC では「コマンド名」以上に、シェルの記号や概念の理解が問われる。
これらは特定レベルに縛らず、Mission の中で自然に登場させる。

| 要素 | 実PCでの意味 | ゲーム内での対応（提案） | LPIC |
|---|---|---|---|
| ワイルドカード `*` `?` `[...]` | ファイル名のパターン一致（glob） | **捜索令状の範囲指定**。「`case_` で始まる全ファイルを一括調査」 | 103.3 |
| 引用符 `'` `"` `\` | 空白・特殊文字の保護、変数展開の制御 | スペース入りの偽装ファイル名（`"top secret.txt"`）を開ける唯一の方法 | 103.1 |
| `2>` / `2>&1` | 標準エラー出力の行き先を変える | 「雑音（エラー）」と「声（結果）」は別の管で流れている、を体感する | 103.4 |
| `/dev/null` | 書き込むと消える特殊ファイル | **シュレッダー**。雑音を捨てて声だけ聞く | 103.4 |
| `<`（入力リダイレクト） | ファイルをコマンドの入力にする | 証拠文書を道具に「読み込ませる」 | 103.4 |
| 終了コード（0=成功） | コマンドの成否を数値で返す仕組み | 部下からの「成功しました(0)/失敗しました(非0)」報告 | 105.2 |
| 環境変数 `PATH` / `HOME` | コマンド探索経路 / ホームの場所 | 道具箱の場所リスト / 自分の事務所の住所 | 103.1 |
| Tab 補完 | 入力途中のファイル名・コマンドを補完 | 相棒の「先読みサポート」。長いパスも一瞬（実機でも同じ、と明示） | — |
| `↑` / `↓`（履歴移動） | 過去の入力を呼び出す | 直前の行動をなぞり直す | 103.1 |
| `Ctrl+R` | コマンド履歴の逆検索 | 記憶（履歴）からの想起。「あのとき何て打ったっけ」 | 103.1 |
| `Ctrl+C` | 実行中コマンドの中断 | 緊急停止ボタン（`tail -f` の張り込みを切り上げる時に必須化） | 103.1 |
| `Ctrl+L` | 画面クリア（`clear` と同じ） | 机の上を一瞬で片付ける | — |
| FHS（`/etc` `/var` `/home` `/tmp` `/bin`） | Linux 標準のディレクトリ構成 | **街の地図**: 市役所(`/etc`)・公文書館(`/var/log`)・住宅街(`/home`)・ゴミ捨て場(`/tmp`)・道具街(`/bin`) | 104.x |
| 正規表現メタ文字 | パターンを記述する言語 | 犯人の「手口のパターン」を書き表す言葉（Mission 判定にも使用中） | 103.7 |
| `man` のセクション番号 | 1=コマンド、5=ファイル形式 など | 捜査ハンドブックの「章立て」（Mission12 で `man 5 crontab` を使用） | — |

**ゲーム設計のポイント:**
Tab 補完・`Ctrl+R`・`↑` はフロントエンド（xterm.js）側の実装。導入すると入力ストレスが激減し、
「実機のターミナルにも同じ機能がある」という発見自体が黒い画面アレルギー克服の成功体験になる。

### 実装難度が高い候補（Phase3 以降・要検討）

| 項目 | LPIC | 課題 |
|---|---|---|
| `vi` / `vim` 基本操作 | 103.8（出題必須領域） | モーダル操作の再現コストが高い。xterm.js 上の簡易 vi（i / ESC / :wq / dd 程度）で「モードの概念」だけ体験させる案 |
| `mount` / `umount` | 104.3 | 仮想FSの多層化が必要。「別の証拠倉庫を接続する」演出案のみ先行メモ |
| パッケージ管理（`apt` / `rpm`） | 102.4/102.5 | ゲーム内では「道具の取り寄せ」。mock ストアで擬似再現する案 |

## 3. ゲーム内機能の提案（コマンド以外）

1. **探偵ランク制度**: § 1 のレベルと連動。Mission クリアでランクアップ演出（辞令 + 新コマンド解放が右パネルに追加される）
2. **コマンド図鑑**: 一度でも正常実行したコマンドが「捜査道具図鑑」に登録され、実PCでの意味とゲーム内の意味が並記で読める（LPIC学習マップの内容をゲーム内に露出する導線）
3. **スマート捜査ボーナス**: 既存の加点（任意コマンド使用）を拡張し、「パイプ1行で解決」「より少ないコマンド数でクリア」にスコアを付与。リプレイ性を作る
4. **相棒キャラクター**: 3段階ヒント（方向性→具体→ほぼ答え）の語り手として相棒を配置。ヒント参照はペナルティなし（挫折させない方針を維持)
5. **隠しファイル収集要素**: 各 Mission に `.secret_memo` 等の隠しファイルを配置。`ls -a` で発見でき、集めると事件の背景ストーリーが読める（クリア条件外の探索動機付け）
6. **`man` = 捜査ハンドブック**: ゲーム内マニュアルを man コマンドで読ませることで、「困ったら man」の実務習慣を植え付ける
7. **タイムアタック演出（見た目のみ）**: 爆弾解除系 Mission の残り時間表示は演出に留め、実際には時間切れ失敗を作らない（state 復元仕様と挫折防止方針との整合のため）
8. **リプレイ台帳**: クリア済み Mission の自分のコマンドログを後から閲覧できる。「あの時どう解いたか」の復習 = LPIC 対策の実効性を高める

## 4. Mission 拡張案集

フォーマットは `Mission参照ファイル.md` § 1 の共通テンプレートに準拠（ここでは案レベルの粒度）。
Mission4/5 は既存の学習テーマ案（パイプ / 権限管理）を具体化したもの。番号は仮。
Mission20「Case Closed」を最終章とし、4〜19 は独立性を保つ（順序の入れ替え・取捨選択が可能）。

### Mission4: Wiretap Tape「盗聴テープを解析せよ」

- **学習テーマ**: パイプ・リダイレクト（Level 5 / LPIC 103.4）
- **あらすじ**: 押収した盗聴記録（数万行のログ）に、犯人が繰り返し連絡していた電話番号が埋もれている。全部読むのは不可能だ。流れ作業で捌け。
- **想定フロー**: `wc -l tape.log`（分量に絶望する演出）→ `head` / `tail` で構造把握 → `grep "TEL" tape.log \| sort \| uniq -c \| sort` で最頻出番号を特定 → 番号を `case_file.sh` に記述 → 疑似 git push
- **必須コマンド**: `grep`, `\|`, `sort`, `uniq`（`wc`, `head`, `tail` は加点）
- **クリア条件案**: `uniq\s+-c` を含むパイプ行の実行 + 正解番号 `TEL: [0-9]{3}-[0-9]{4}` の記述
- **ゲーム性**: 「cat では読み切れない」体験そのものがパズル。パイプの必要性を強制的に体感させる

### Mission5: The Locked Vault「開かずの資料室」

- **学習テーマ**: パーミッション（Level 7 / LPIC 104.5）
- **あらすじ**: 重要証拠は資料室にあるが、ファイルには鍵（権限）がかかっている。読めない・開けない・実行できない、3種類の鍵を使い分けろ。
- **想定フロー**: `ls -l` で `----------` を確認 → `chmod +r` で閲覧解錠 → 中のヒントから次の部屋へ → 実行権限のない `case_file.sh` を発見 → `chmod +x` → 実行してクリア
- **必須コマンド**: `ls -l`, `chmod`, `cat`
- **クリア条件案**: `chmod\s+\+?[rx]` 系パターン2回以上 + case_file 実行成功
- **ゲーム性**: `rwx` 表示を「鍵の刻印」として読み解く。数値モード（`chmod 644`）での解錠は加点

### Mission6: Shadow Process「盗聴器を止めろ」

- **学習テーマ**: プロセス管理（Level 6 / LPIC 103.5)
- **あらすじ**: 事務所の中に盗聴プログラムが仕掛けられている。動いているプロセスを調べ、正体を突き止めて止めろ。ただし、間違って時計やポストを止めるな。
- **想定フロー**: `ps aux` で一覧 → 正規プロセス（`clock`, `mailbox`, `heater`）に混ざる `listener_x` を発見 → `cat /proc_notes/listener_x.txt` 等で裏取り → `kill <PID>` で停止
- **必須コマンド**: `ps`, `kill`
- **クリア条件案**: 正しい PID への kill 実行 + `mission_flags.bug_removed = true`
- **ゲーム性**: 「間違い探し」。正規プロセスを kill すると警告メッセージ + 巻き戻し（即失敗にしない）。裏取りせず勘で kill すると正解でも減点、という「証拠を掴んでから動く」探偵らしさを評価

### Mission7: Master of Disguise「変装潜入」

- **学習テーマ**: ユーザー切替（Level 7 / LPIC 107.1）
- **あらすじ**: 容疑者「barman」しか読めないファイルがある。合言葉（パスワード）は店のどこかに。変装して読め。ただし変装中の自分を見失うな。
- **想定フロー**: `cat` で `Permission denied` を体験 → 店内探索で合言葉発見 → `su barman` → `whoami` で確認 → 目的ファイル閲覧 → `exit` で自分に戻る
- **必須コマンド**: `su`, `whoami`, `cat`, `exit`
- **クリア条件案**: barman 状態での対象ファイル閲覧 + 元ユーザーへの復帰
- **ゲーム性**: 「今の自分は誰か」を `whoami` で確認する習慣づけ。exit で戻る構造が Mission3 の ssh/exit と対になっており、「セッションに入る/出る」概念が反復学習になる

### Mission8: Sealed Evidence「封印された証拠品」

- **学習テーマ**: アーカイブ・鑑識（Level 8 / LPIC 103.3）
- **あらすじ**: 押収品はマトリョーシカのように何重にも封印されている（`evidence.tar.gz` の中に `inner.zip`、その中に…）。しかも拡張子は当てにならない。鑑識（file）にかけながら開封しろ。
- **想定フロー**: `file evidence.dat`（実体は tar.gz と判明）→ `tar -xzf` → 出てきたファイルをまた `file` → `unzip` → … → 最深部の手がかり入手
- **必須コマンド**: `file`, `tar`, `gunzip` または `unzip`
- **クリア条件案**: 最深部ファイルの `cat` 実行 + 記載コードの report 転記
- **ゲーム性**: 開封のたびに一歩近づく「発掘」感。「拡張子ではなく file で確かめる」という実務の鉄則をパズル化

### Mission9: The Forged Letter「改ざんされた遺言状」

- **学習テーマ**: 差分・置換（Level 5 / LPIC 103.2）
- **あらすじ**: 遺言状の原本と提出された写しがある。どこかが書き換えられている。差分を見つけ、写しを原本どおりに直せ。
- **想定フロー**: `diff original.txt submitted.txt` で改ざん行を特定 → `sed 's/偽の相続人/正しい相続人/' submitted.txt` で復元 → 復元結果を report へ
- **必須コマンド**: `diff`, `sed`
- **クリア条件案**: diff 実行 + sed による正しい置換 + 復元後ファイルの一致判定
- **ゲーム性**: 「間違い探し」を機械にやらせる快感。目視では見つからない1文字差（`0` と `O`）を仕込む

### Mission10: Torn Note「切り裂かれた脅迫状」

- **学習テーマ**: テキスト整形（Level 5 / LPIC 103.2）
- **あらすじ**: 脅迫状は細切れにされてゴミ箱に捨てられていた。断片ファイル（`piece_03.txt` など行番号付き）を並べ替え、貼り合わせて全文を復元しろ。
- **想定フロー**: `ls` で断片確認 → `cat piece_*` で中身把握 → `sort` で行順復元 → `cut` で行番号を除去、または `paste` で結合 → 全文を report へ
- **必須コマンド**: `sort`, `cut` または `paste`
- **クリア条件案**: 復元済み全文の正規表現一致
- **ゲーム性**: 物理的な「復元作業」をコマンドで再現。手作業（1個ずつ cat して echo 転記）でも解けるが、sort/cut を使うと圧倒的に速い、という「道具の価値」を体験させる

### Mission11: Ghost Line「幽霊回線を追え」

- **学習テーマ**: ネットワーク追跡（Level 9 / LPIC 109.x・110.1）
- **あらすじ**: 犯人は通称「ghost」というサーバーから指示を出している。名前しか分からない。住所（IP）を割り出し、生存を確認し、突入せよ。
- **想定フロー**: `dig ghost.example`（IP 判明）→ `ping` で生存確認 → `ss` で開いている扉（ポート）確認 → `ssh ghost.example` → remote 内で証拠収集 → `exit`
- **必須コマンド**: `dig`（または `host`）, `ping`, `ssh`, `exit`
- **クリア条件案**: dig → ping → ssh の順序実行 + remote 内の証拠ファイル閲覧
- **ゲーム性**: Mission3 の ssh 体験の発展形。「調べてから踏み込む」実務手順が捜査手順そのものとして機能する

### Mission12: Midnight Broadcast「深夜0時の犯行予告」

- **学習テーマ**: cron・時刻書式（Level 10 / LPIC 107.2）
- **あらすじ**: 犯人はサーバーに時限装置（cron ジョブ）を仕掛けた。`0 0 * * 5` …この暗号はいつ発動する？ 解読して、正しいジョブだけを解除しろ。
- **想定フロー**: `crontab -l` でジョブ一覧 → 5フィールドの時刻書式を読み解く（ヒント: 捜査ハンドブック `man 5 crontab`）→ 危険ジョブを特定 → 解除コマンド実行 → 解除時刻を report へ
- **必須コマンド**: `crontab -l`, `cat`, `date`
- **クリア条件案**: 正しいジョブの特定（発動日時の記述一致）+ 解除フラグ
- **ゲーム性**: cron 書式の読解を「時限爆弾の暗号解読」に変換。LPIC 頻出の暗記項目（分 時 日 月 曜）が、暗記ではなく解読体験として定着する

### Mission13: Hall of Mirrors「鏡の館」

- **学習テーマ**: リンク・実体判定（Level 8 / LPIC 104.6）
- **あらすじ**: 証拠品保管庫は「鏡の館」だった。同じ名前のファイルがいくつもあるが、ほとんどは案内板（シンボリックリンク）で、実体は1つだけ。本物を見つけ出せ。
- **想定フロー**: `ls -l` で `->` 表示を読む → リンクを辿る/`file` で `symbolic link` 判定 → 実体ファイルを特定して閲覧 → 実体の絶対パスを report へ
- **必須コマンド**: `ls -l`, `file`
- **クリア条件案**: 実体ファイルの絶対パス記述（リンクのパスは不正解）
- **ゲーム性**: 「どれが本物か」の推理がそのまま `ls -l` の読解訓練になる。Mission2 の絶対パス学習の応用編

### Mission14: The Informant's Trail「情報屋の足取り」

- **学習テーマ**: 履歴・ログ監視（Level 5〜6 / LPIC 103.1）
- **あらすじ**: 情報屋が失踪した。彼の端末には操作履歴（history）が残っている。最後に何を調べ、どこへ向かったのか。足取りを再現しろ。
- **想定フロー**: `history`（情報屋の残した履歴を閲覧する演出）→ `tail journal.log` で最終記録確認 → 履歴のとおりに自分でも同じコマンドを再実行 → 行き先を突き止める
- **必須コマンド**: `history`, `tail`, `grep`
- **クリア条件案**: 履歴再現（同一コマンド列の実行）+ 行き先の記述一致
- **ゲーム性**: 「他人のコマンド履歴を読んで意図を推理する」= コマンドの意味理解の総復習。履歴がそのまま攻略チャートになるメタ構造

### Mission15: The Great Sweep「一斉捜索令状」

- **学習テーマ**: ワイルドカード・引用符（レベル外・概念 / LPIC 103.1・103.3）
- **あらすじ**: 押収品倉庫には数百のファイル。令状が許すのは「`case_` で始まり数字1桁で終わるファイル」だけだ。範囲をパターンで指定して一括捜索しろ。なお犯人は、ファイル名に空白を仕込んだ目くらまし（`top secret.txt`）を残している。
- **想定フロー**: `ls case_*`（多すぎる）→ `ls case_[0-9].txt` で令状範囲に絞る → 該当ファイルを `grep` 一括調査 → `cat top secret.txt` が `Error: invalid input` になる → `cat "top secret.txt"` で開封成功
- **必須コマンド**: glob（`*` `?` `[...]`）を含むコマンド実行、引用符付きパス
- **クリア条件案**: `[0-9]` 等の glob パターン使用 + 引用符付きファイルの閲覧 + 発見コードの report 記述
- **ゲーム性**: 「令状の範囲＝パターン」という対応で glob を学ぶ。空白入りファイル名は「引用符を知らないと物理的に開けない」ため、詰まる→ヒント→開けた時の納得感が強い

### Mission16: Fingerprint「指紋は嘘をつかない」

- **学習テーマ**: チェックサム・真正性検証（Level 8 追補 / 実務必須）
- **あらすじ**: 金庫から見つかった契約書のコピーが5通。見た目は完全に同一だが、1通だけ改ざんされている。目視では見抜けない。ファイルの「指紋」で照合しろ。
- **想定フロー**: `cat copy_*.txt`（全部同じに見える）→ 台帳 `ledger.txt` に原本のハッシュ値がある → `md5sum copy_*.txt` → 台帳と突き合わせて不一致の1通を特定 → `diff` で改ざん箇所を確認 → report へ
- **必須コマンド**: `md5sum`（または `sha256sum`）, `cat`（`diff`, `sort` は加点）
- **クリア条件案**: md5sum 実行 + 改ざんファイル名と改ざん箇所の記述一致
- **ゲーム性**: 「1文字でも違えば指紋は別物になる」というハッシュの性質を推理トリックとして使う。Mission9（diff）の発展形で、「差分を探す」から「同一性を証明する」へ視点が上がる

### Mission17: Silence in the Static「雑音の中の声」

- **学習テーマ**: 標準エラー出力・リダイレクト・終了コード（レベル外・概念 / LPIC 103.4・105.2）
- **あらすじ**: 老朽化したアーカイブサーバーは、探索のたびに大量のエラー（雑音）を吐く。本当の手がかり（声）は雑音に埋もれて読めない。雑音だけをシュレッダーに流せ。
- **想定フロー**: `grep -r "witness" /archive` → 画面が `Permission denied` の雑音で埋まる → `grep -r "witness" /archive 2>/dev/null` で声だけが残る → 手がかり入手 → `echo $?` で直前の捜査の成否（0/1）を確認する練習
- **必須コマンド**: `2>/dev/null` を含むコマンド実行（`echo $?` は加点）
- **クリア条件案**: `2>\s*/dev/null` パターンの実行 + 手がかりの report 記述
- **ゲーム性**: 「出力には2本の管がある」という LPIC 頻出概念を、雑音と声の演出でそのまま可視化する。雑音で画面が流れる絶望→1記号で静寂、のカタルシス

### Mission18: The Detective's Playbook「捜査手順書を書け」

- **学習テーマ**: シェルスクリプト基礎（Level 11 / LPIC 105.1・105.2）
- **あらすじ**: お前も後輩を持つ身になった。これまで手作業でやってきた「証拠確認→判定→報告」を、誰でも実行できる手順書（スクリプト）に書き起こせ。今まで実行するだけだった `case_file.sh`、今日からは書く側だ。
- **想定フロー**: `cat /root/desk/sample.sh` で見本を読む → `echo` で自作 `patrol.sh` を作成（`TARGET="..."` の変数定義、`if grep -q "$TARGET" file; then echo "FOUND"; fi`）→ `sh patrol.sh` で実行 → `FOUND` 出力を確認
- **必須コマンド**: `sh`, 変数定義と `$変数` 参照, `if` 文
- **クリア条件案**: 変数定義 + if 文を含むスクリプトの作成と実行成功
- **ゲーム性**: 「実行する側→書く側」への視点逆転が成長の実感になる。自分の書いた手順書が動く瞬間は、プログラミング入門の原体験そのもの。for 文で複数ファイルを巡回すると加点

### Mission19: Map of the City「この街の地図」

- **学習テーマ**: FHS・実 Linux のディレクトリ構成（レベル外・概念 / LPIC 104.x）
- **あらすじ**: 最終決戦を前に、舞台は探偵事務所から「本物の Linux の街」へ。設定はすべて市役所（`/etc`）に、出来事はすべて公文書館（`/var/log`）に、住民は住宅街（`/home`）に、消したいものはゴミ捨て場（`/tmp`）にある。街の地図を頭に入れ、黒幕の住民登録を探し出せ。
- **想定フロー**: `ls /` で街の全体図 → `cat /etc/hosts`（知り合いの住所録）→ `/var/log/entry.log` を `tail` / `grep` で出入り記録調査 → `/tmp` に消し忘れた痕跡 → `/home/<黒幕>` を特定
- **必須コマンド**: `ls`, `cd`, `cat`, `grep`, `tail`
- **クリア条件案**: `/etc` `/var/log` `/tmp` `/home` の4区画すべての探索フラグ + 黒幕ユーザー名の記述一致
- **ゲーム性**: 新コマンドゼロ・知識だけで解く「地理」の回。ゲーム終盤で実機と同じ地形に慣れておくことで、クリア後に本物のターミナルを開いた時「知っている街」になっている

### Mission20: Case Closed「最終事件 — すべてを繋げろ」

- **学習テーマ**: 総合演習（Level 1〜11 の複合）
- **あらすじ**: これまでの全事件は一人の黒幕に繋がっていた。local と複数の remote を行き来し、権限を解除し、ログをパイプで捌き、封印された最終証拠の真正性をハッシュで確かめ、黒幕の名を突き止めて本部に提出せよ。
- **想定フロー**: 各 Mission で学んだ技術を1つずつ関所として配置（find → dig/ssh → chmod → grep\|sort\|uniq → tar → md5sum → 自作スクリプトで最終判定 → 報告）
- **必須コマンド**: `find`, `ssh`, `chmod`, `grep`, `\|`, `tar`, `md5sum`, `sh`（自作）, 疑似 git 一式
- **クリア条件案**: 全関所フラグの達成 + 黒幕名の正規表現一致 + git push
- **ゲーム性**: 卒業試験。ヒントは1段階目のみ（相棒が「もう教えることはない」と言う演出）。クリア時に「実PCでも同じことができる」旨のエピローグを表示し、実機・LPIC 受験への導線とする

## 5. 導入時に必要な作業（採用が決まったら）

### allowlist / denylist の更新（`設計指示書.md` § 8）

| レベル | allowlist への追加候補 |
|---|---|
| Level 5 | `head`, `tail`, `wc`, `cut`, `paste`, `tr`, `sed`, `diff`, `nl`, `tee`, `xargs` + パイプ `\|` の構文許可 |
| Level 6 | `ps`, `top`, `kill`, `pgrep`, `jobs`, `free`, `uptime` |
| Level 7 | `umask`, `su`, `whoami`, `id`, `who` （`chmod` / `chown` は既存） |
| Level 8 | `tar`, `gzip`, `gunzip`, `zip`, `unzip`, `ln`, `file`, `which`, `locate`, `du`, `df` |
| Level 9 | `ping`, `ip`, `ss`, `dig`, `host`, `hostname`, `traceroute` |
| Level 10 | `crontab`, `at`, `date`, `cal`, `systemctl`, `journalctl`, `uname`, `env`, `alias` |
| Level 11 | `sh`, `test`, `read`, `basename`, `dirname`, `seq` |
| Level 8 追補 | `md5sum`, `sha256sum`, `stat` |
| 共通 | `man`, `apropos`, `whatis` |

**構文レベルの許可（コマンド名ではなく evaluator のパーサー対応）:**
glob（`*` `?` `[...]`）/ 引用符 / `2>` `2>&1` `<` リダイレクト / `$?` / 変数定義・展開 / `if` `for` `test`

**要注意（denylist との整合）:**
- `kill` は現行どちらのリストにもない → 仮想プロセス限定で allowlist へ（実プロセスには一切触れない）
- `sudo` は denylist 維持。導入する場合もイベント演出限定とし、自由実行は許可しない
- `dd` / `mkfs` / `rm` は引き続き denylist（変更しない）
