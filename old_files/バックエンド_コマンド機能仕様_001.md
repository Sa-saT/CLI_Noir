# バックエンド コマンド機能仕様（MVP）

evaluator 実装時の書き漏れを防ぐためのコマンド定義書。  
allowlist/denylist 定義・エラーメッセージ定義は `設計指示書.md` § 8, § 12 を正とする。

---

## 0. 実装ルール（共通）

- 判定順序: `denylist → allowlist → コマンド実行`（`設計指示書.md` § 8 参照）
- `git` は第1トークンで判定後、サブコマンドを別途分岐する
- エラー文言は `設計指示書.md` § 12 の一覧を使う
- 不正コマンドは即失敗にせず `Error: command not allowed` を返す

---

## 1. コマンド実装テンプレート（必須項目）

各コマンドは最低限、以下を記述してから実装すること。

- コマンド名 / サブコマンド
- 引数形式
- 前提条件（local/remote、対象存在、権限など）
- 正常時出力
- 異常時出力
- 更新する state
  - `current_path`
  - `filesystem`
  - `remote_mode`, `ssh_host`
  - `git_state`
  - `mission_flags`
- Mission判定への影響

---

## 2. 基本コマンド

### `ls [path]`
- 前提: `path` 未指定時は `current_path`
- 正常: 対象ディレクトリ配下を一覧表示
- 異常: 対象が存在しない場合 `Error: path not found`
- state更新: なし

### `cd <path>`
- 前提: 移動先がディレクトリとして存在する
- 正常: `current_path` を更新
- 異常: `Error: directory not found`
- 制約: remote から local へは `cd` で戻れない（`exit` のみ）

### `pwd`
- 正常: `current_path` を表示
- state更新: なし

### `touch <file>`
- 正常: ファイル生成（未存在時）
- 異常: 親ディレクトリ不正時 `Error: path not found`
- state更新: `filesystem`

### `mkdir <dir>`
- 正常: ディレクトリ生成
- 異常: 既存名の場合 `Error: directory already exists`
- state更新: `filesystem`

### `cat <file>`
- 正常: ファイル内容を返す
- 異常: `Error: file not found`
- state更新: なし

### `less <file>`
- 正常: ページング表示（MVP は全体返却でも可）
- 異常: `Error: file not found`
- state更新: なし

### `echo ...`
- 正常: 文字列表示またはリダイレクト先へ追記
- 異常: リダイレクト先の親パス不正時 `Error: path not found`
- state更新: `filesystem`（追記時）

### `history`
- 正常: コマンド履歴表示
- state更新: なし

### `clear`
- 正常: ターミナル表示クリア
- state更新: なし

---

## 3. 検索/整形コマンド

### `grep <pattern> <file>`
- 正常: 一致行を返す
- 異常: ファイルなし `Error: file not found` / パターン不正 `Error: invalid pattern`

### `find <path> ...`
- 正常: 条件一致ファイルを返す
- 異常: パス不正 `Error: path not found`

### `sort`
- 正常: 入力行を昇順で返す
- 異常: 入力不正時 `Error: invalid input`

### `uniq`
- 正常: 重複行をまとめて返す
- 異常: 入力不正時 `Error: invalid input`

### `awk <expr> <file>`
- 正常: 条件一致結果を返す
- 異常: 式不正 `Error: invalid awk expression` / ファイルなし `Error: file not found`

---

## 4. SSH / Remote

### `ssh <host>`
- 正常: `remote_mode=true`, `ssh_host=<host>`, `current_path=<remote初期パス>`
- 異常:
  - 未登録ホスト: `Host not found`
  - 権限不可: `Permission denied`

### `exit`
- 正常: local に戻す（`remote_mode=false`, `ssh_host=null`, `current_path=/root`）
- 異常: local で実行時は no-op またはガイド文言

## 5. 疑似Git

`git` はサブコマンドで分岐する:
- `git status`
- `git add`
- `git commit -m "<msg>"`
- `git push`

### `git status`
- 正常: `git_state` に応じて
  - `Nothing to commit`（変更なし）
  - `Changes staged`（add 済み）
  - `No commits yet`（add はあるが commit 未）
  - `Ready to push`（commit 済み + case_checked）

### `git add`
- 正常: 変更ファイル + mission_flags の現在値を staged に上げる
- state更新: `git_state.staged`

### `git commit -m`
- 前提: staged あり + message 1文字以上
- 正常: 現在の state snapshot（current_path / filesystem / mission_flags）を commits 配列に追加
- 異常:
  - staged なし: `Error: nothing to commit`
  - message なし: `Error: commit message required`
- state更新: `git_state.commits` に新規 commit を push
- 1 Mission 中に何度でも実行可（複数セーブポイント）

### `git push`
- 前提: commits が 1 件以上存在する
- 正常: 最新 commit の snapshot を判定し、`case_checked=true` なら `pushed=true` → ミッション完了
- 異常:
  - commit なし: `Error: push not allowed before commit`
  - クリア条件未達: `Error: mission requirements not met`
- 次 Mission 遷移時、前 Mission のクリア前 commit は全消去される

## 6. システム補助コマンド

### `chmod`, `chown`, `curl`
- MVP方針: 実OS操作は行わない
- `curl` は mock API 呼び出し用途に限定し、外部ネットアクセスは禁止
