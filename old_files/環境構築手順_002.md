# CLI_Noir 環境構築手順（インストール済み前提）

このドキュメントは、`volta` / `pnpm` / `brew` / `pyenv` が既にインストール済みの状態から、  
CLI_Noir の開発環境を立ち上げるための実行手順をまとめたものです。

---

## 1. 前提

- OS: macOS
- シェル: zsh
- `volta`, `pnpm`, `brew`, `pyenv` はインストール済み

---

## 2. フロントエンド構築（Vue3 / Nuxt）

### 2-1. Node / pnpm バージョン確認
```bash
node -v
pnpm -v
volta list
```

### 2-2. Nuxt プロジェクト作成
```bash
pnpm create nuxt@latest frontend
cd frontend
pnpm install
```

### 2-3. 開発サーバー起動
```bash
pnpm dev
```

---

## 3. バックエンド構築（Python / Django REST）

### 3-1. Python バージョン固定
```bash
pyenv local 3.12.8
python -V
```

### 3-2. 仮想環境作成と有効化
```bash
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
```

### 3-3. 依存パッケージ導入
```bash
pip install django djangorestframework djangorestframework-simplejwt "channels[daphne]"
```

- `djangorestframework-simplejwt`: JWT の発行/検証（PyJWT を直接組まない。2026-07-06 改訂）
- `channels[daphne]`: Channels 4 は daphne を入れないと `runserver` が WebSocket を受けられない

### 3-4. Django プロジェクト作成
```bash
django-admin startproject backend .
python manage.py startapp missions
```

### 3-5. 起動確認
```bash
python manage.py migrate
python manage.py runserver
```

---

## 4. 推奨ディレクトリ構成（例）

```text
project-root/
├ frontend/            # Nuxt (Vue3)
├ backend/             # Django project
├ missions/            # Mission仕様ドキュメント
└ docs/                # 設計・運用ドキュメント
```

---

## 5. 動作確認チェック

### フロント
- `pnpm dev` で起動できる
- `/missions` ルートへアクセスできる

### バック
- `python manage.py runserver` で起動できる
- `/api/auth/login/` と `/api/missions/` のエンドポイントが定義済み

---

## 6. トラブルシュート

- `ModuleNotFoundError`
  - 仮想環境有効化（`source .venv/bin/activate`）後に `pip install` を再実行する

- Python バージョンが想定と異なる
  - `pyenv local 3.12.8` を再実行し、`python -V` を確認する

---
