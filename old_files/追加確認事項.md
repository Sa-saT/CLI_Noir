## 9) 受け入れ基準（明文化）
- 前提: Mission1〜3がMVP。
- 確認したいこと:
  - Mission1〜3の完了条件（1行で可）
  - 主要コマンドの正常系/異常系テストケースの粒度
- 回答:
  - Mission1:
  机の上の名刺ファイルに自分のユーザー名を書き込んで正しい case_file.sh を実行し、その後 git push が成功すること。
  - Mission2:
  公園ディレクトリ内のヒントファイルを探索し、猫ファイルの絶対パス・正しい状態を含む case_file.sh を作成して実行し、git push が成功すること。
  - Mission3:
  遊園地（remote）内の各遊具ディレクトリからヒントを収集して爆弾.zip を展開し、必要情報を正規表現で満たした case_file.sh を作成・実行し、git push が成功すること。

  - テストケース粒度:
  以下は 主要コマンド や 機能ごとにテストすべき粒度 の目安です。
  実装時のテスト設計や QA シナリオとしてそのまま使えます。

    1) 基本ターミナル操作系
    ls:存在するパスを一覧表示。存在しないパスはエラー表示。
    cd:有効なディレクトリへ移動。存在しないディレクトリはエラー。
    pwd:現在パスを正しく表示。
    history:コマンド履歴を出力。
    clear:画面をクリア。

    ▶ 基本操作群はフルパスで検査
      •	ls /root/desk
      •	cd park
      •	pwd

    2) ファイル操作系
    touch:新規ファイルの作成。ファイル名不正でエラー。
    mkdir:ディレクトリ作成。既存ディレクトリエラー。
    cat:ファイル内容表示。存在しないファイルでエラー。
    less:ファイルページ表示。存在しないファイルでエラー。
    echo:出力追記/表示。引数なしやパス不正。

    ▶ 異常系は結果メッセージも検証
      $ cat nonexist.txt
      Error: file not found

    3) 検索・テキスト処理
    grep:正規表現に一致する行を表示。マッチなしは空結果。
    find:条件に一致するファイルを返す。不正なパスでエラー。
    sort:正しくソート出力。空データでも動作。
    uniq:ソート済みで重複排除。不整合データは空出力。
    awk:正規表現・条件判定。パターン不正でエラー。

    4) SSH / Remote
    ssh target:remote_mode に遷移。存在しない target で “Host not found”。
    exit:local 移動。local で exit は ignore。

    5) Git 風ワークフロー
    git status:現在ステータス表示。無変更でも動作。
    git add .:変更全追加。追加対象なしでも OK。
    git commit -m:正常コミット。message が無いとエラー。
    git push:次のミッションへ遷移。条件不満足でエラー。
    ※内部git_stateに基づいて表示。

    6) エラー/警告系
    許可外コマンド:“Error: command not allowed”
    正規表現不一致:“Error: pattern mismatch”
    正常ではあるがパターン不一致結果:“Warning: pattern mismatch”
    remote 未接続で remote だけのファイルアクセス:“Error: remote not connected”“Command not available in local mode” など

    正規表現評価
      •	完全一致 / 部分一致のテスト
      •	空白・余分スペースの除去
      •	大小文字区別の確認
      grep "^Height: [0-9]+$" catinfo.txt

    ▶ 正規表現評価はパターン一致結果を検証
      Expected Height: 150

    仮想FS シナリオ
      •	local と remote の切り替え
      •	exit で local 戻り
      •	cd で remote → local へは戻せない

    ミッション進行
      •	正しい順序で Git 操作ができるか
      •	誤った順序では進まないこと

    ▶ ミッション単位
      •	スクリプト正常
      •	正規表現一致
      •	Git ワークフロー成功
